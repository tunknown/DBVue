Тип поля

P/U/M/F/L/J/D

0-1

P= локатор(первичный ключ) в гриде (в т.ч. по нескольким полям) при редактировании;

U= поле, доступное для лукапа из другого места, дополнительно к P;

M= FK-мастер (в т.ч. по нескольким полям). Название группы нужно, если внешний ключ из нескольких полей смотрит на один и тот же объект. Обычно, без L/J-лукапа;

F= FK-лукап (в т.ч. по нескольким полям). Название группы нужно, если лукап из нескольких полей смотрит на один и тот же объект. Обычно, дополняется L/J-лукапом;

L= Name-лукап, при TField.FieldKind=fkLookup в виде DropDown (не хранится);

J= Name-лукап, при TField.FieldKind=fkData через join в виде cbElipsys (не хранится);

D= неинтерпретируемые данные, можно не указывать- по умолчанию
