Схема объекта только для разделения по пользователям, если в ней есть бизнес-смысл, то её в подкласс через разделитель «.»

Имя объекта уникально во всех схемах, это не позволяет отменять установки администратора, например, по порядку показа и выбору по-умолчанию


Классы объектов могут содержать подклассы через разделитель «.»

Что означает незаполненный подкласс «*..*» при заполненном подподклассе?

Класс/атрибут поддерживают наследование и полиморфизм. Например, действие по « Contragent » доступно для объекта « Contragent.Source » , но не наоборот.

Для уникализации класс можно заканчивать на «.число» если наследование не требуется.


Изменение классов объектов затруднено, т.к. они захардкодены без жёстких FK во всех объектах и их затратно перепривязывать.

N= для L/J-лукапа из другого места(design-time, в run-time не используется), желательно, уникальное для выбора пользователем;

M= в т.ч. по нескольким полям. Название группы нужно, если внешний ключ из нескольких полей смотрит на один и тот же объект. Обычно смотрит на B/V без L/J-лукапа. F= в т.ч. по нескольким полям. Название группы нужно, если лукап из нескольких полей смотрит на один и тот же объект. Обычно смотрит на V-справочник с L/J-лукапом. Не попадает в динамическую часть фильтрации;
